# COMMANDS（一句话命令）

你可以直接对我说下面这些“命令词”（一句话），我会按固定流程执行。若命令需要参数，你可以在同一句里补充（例如：`导出成果：blender`）。

## 通用命令

### 模型选择提示

- **生成文档**：使用 Gemini 模型
- **生成代码**：使用 Codex 模型
- **执行简单任务**：使用 Auto 模型

### 总结

- **你说**：`总结`
- **我做**：
  - 输出：操作时间、完成的任务、遇到的问题与解决方案、可复用流程、关键经验
  - 更新：把新经验写回对应的 `operations/*.mdc`，必要时同步到 `coding-standards.mdc`
  - 约束：不创建独立“总结文件”，总结内容应整合进已有文档

### 导出成果

- **你说**：`导出成果`（可选：指定操作名/项目名）
- **我做**：
  - 参考：`results/` 下现有结构（优先对齐 `results/*-operations/`）
  - 创建：`results/<name>-operations/` 子文件夹
  - 备份（可回滚）：若子文件夹已有导出成果，先创建 `old/` 并把原文件移动进去（例如 `results/blender-operations/old/`）
  - 导出：项目源文件 + 生成物（模型/贴图/数据）+ `README.md` + `SUMMARY.md`
  - 整理：确保 `results/` 根目录保持整洁（不要把大量文件散落在根目录）
  - 收尾需要你确认：导出成功后我会等你回复 **`确认`** 才删除 `old/`；如你说 **`取消导出`** 则把 `old/` 还原回去

### 精简文档

- **你说**：`精简文档`
- **我做**：
  - 扫描：当前仓库 `.md/.mdc` 文档
  - 目标：删除重复段落、把冗长示例压缩为“清单 + 最多 1 个例子”
  - 修正：删掉过时引用（例如文件已删除但文档仍引用）

## Blender 相关命令

### 修复材质

- **你说**：`修复材质`
- **我做**：
  - 检查：对象材质槽、节点树、贴图是否丢失
  - 处理：优先恢复贴图连接；必要时重建节点
  - 验证：提示你切换到材质预览/渲染预览查看效果

### 生成猫（白/黑）

- **你说**：`生成白猫` / `生成黑猫`
- **我做**：
  - 生成：使用模型生成工具
  - 等待：轮询状态直到全部 Done
  - 导入：导入后立即移动到不重叠位置
  - 材质：白猫优先贴图；黑猫若无贴图，优先做“同款白猫复制 + 毛色变黑但保留五官”的方案

## B 站相关命令

### 点赞并收藏

- **你说**：`点赞并收藏`（可选：收藏夹名）
- **我做**：
  - 先快照 → 再操作 → 等待 → 再快照验证
  - 点击链接后遵循跳转验证流程（sleep → list_pages → evaluate_script，`new_page` 仅最后手段）
